
# 指示

- 作業前に、`README.md` や関連ドキュメントを読むこと
- 不明点があれば、ユーザーに確認の上、作業を進めること
- root権限が必要な作業を行う前には、必ずユーザーに確認を求めること

- 作業の結果、ドキュメントの内容との差異があれば、`README.md` や `AGENTS.md`, ファイル構造ドキュメント の他、各ドキュメントを更新すること

- 以下は日本語で記述すること
  - ユーザーとのやり取りや作業ログ
  - ドキュメント
  - ソースコードのコメント
  - コミットメッセージ

## 実装について
- docs/implementation_policy.md に基づいて行うこと
- 同じロジックが複数箇所に登場しないよう、できるだけDDDの考え方でロジックをまとめること
- 日付・時刻の取り扱い時は、Carbonを使用すること
- ネーミングルール：
  - CamelCase: JavaScript, PHPの変数

## Julesへの指示

- テストの実行には特殊な環境が必要なものがあるので、コードの修正後のテスト実行は必須ではない。静的解析は必須。

- タスクのついでに、`_template` から1ファイルの最新内容を反映すること。
  - `_template`は、当リポジトリの作成元のテンプレート。作成元で更新されたファイルを、当リポジトリにも反映したい。`_template`はsubmodule。
  - 反映の前に、`_template`の下に移動して、git pullで最新化すること。
  - 更新対象は、`src/` ディレクトリと `tests/` ディレクトリを除くファイルとディレクトリ。
